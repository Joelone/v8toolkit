20 /**
21 * Design Questions:
22 * - When a c++ object returns a new object represented by one of its members, should it
23 * return the same javascript object each time as well?
24 * class Thing {
25 * OtherClass other_class;
26 * OtherClass & get_other_class(){return this->other_class;} <== should this return the same javascript object on each call for the same Thing object
27 * }
28 * - that's currently what the existing_wrapped_objects map is for, but if a new object
29 * of the same time is created at the same address as an old one, the old javascript
30 * object will be returned.
31 * - how would you track if the c++ object source object for an object went away?
32 * - how would you actually GC the old object when containing object went away?
33 * - Maybe allow some type of optional class customization to help give hints to V8ClassWrapper to have better behavior
34 *
35 */
